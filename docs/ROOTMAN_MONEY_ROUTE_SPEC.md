# ROOTMAN MONEY ROUTE

## ระบบวิเคราะห์เส้นทางสร้างรายได้ที่เหมาะกับแต่ละคน

> **หมายเหตุเอกสาร:** เวอร์ชันนี้เป็นสเปกที่ **ตัดส่วนที่พึ่งพา AI ออกทั้งหมด**
> ระบบทั้งหมดทำงานด้วย **Deterministic Logic + Template-based Content** ที่ควบคุมได้เต็มที่
> ส่วนที่เคยเป็น AI (การสร้างรายงาน, ผู้ช่วย ROOTOWL, Structured Output, Prompt) ถูกแทนที่ด้วยระบบกฎ (rules) และเทมเพลตเนื้อหาที่ Admin จัดการได้

**Tagline หลัก**

> คุณไม่ได้หาเงินไม่เก่ง
> คุณอาจแค่กำลังเล่นเกมที่ไม่เหมาะกับตัวเอง

**คำอธิบายผลิตภัณฑ์**

ROOTMAN MONEY ROUTE คือ Web Application ภายใต้แบรนด์ ROOTMAN ที่ช่วยวิเคราะห์ว่า ผู้ใช้งานควรสร้างรายได้เพิ่มด้วยวิธีใด โดยพิจารณาจากเงินทุน เวลาว่าง ทักษะ บุคลิก ความสามารถในการขาย ความกล้าเปิดเผยตัวตน ความเสี่ยงที่รับได้ และเป้าหมายด้านรายได้

ระบบไม่ได้จบแค่การบอกว่า "คุณเป็นคนประเภทไหน" แต่จะสร้างเส้นทางปฏิบัติที่ประกอบด้วย:

- เส้นทางสร้างรายได้ที่เหมาะที่สุด
- เส้นทางที่ควรหลีกเลี่ยง
- โมเดลรายได้ที่แนะนำ
- รายการสินค้าและบริการที่สามารถเริ่มทำได้
- แผนทดลองตลาด
- แผนสร้างรายได้ก้อนแรก
- เครื่องมือและ Template พร้อมใช้
- ระบบติดตามความคืบหน้า
- ระบบคำแนะนำแบบกฎ (Rule-based Guidance) ที่ช่วยปรับแผนให้เข้ากับผู้ใช้

# 1. บริบทของแบรนด์ ROOTMAN

ROOTMAN เป็นแบรนด์ความรู้เกี่ยวกับการตลาด การขาย ธุรกิจ จิตวิทยาผู้บริโภค และพฤติกรรมมนุษย์

ภาพลักษณ์ของแบรนด์คือ:

- คม
- ลึก
- ตรง
- ไม่ขายฝันแบบไร้เหตุผล
- เปิดเผยด้านจริงของธุรกิจ
- เข้าใจแรงขับด้านความโลภ ความกลัว ความทะเยอทะยาน และความต้องการมีชีวิตที่ดีขึ้น
- ใช้ภาษาที่คนทั่วไปเข้าใจได้ แต่มีกรอบคิดลึกกว่าคอนเทนต์ตลาดทั่วไป

ROOTMAN MONEY ROUTE ต้องไม่ดูเหมือนแบบทดสอบบุคลิกภาพน่ารักทั่วไป ไม่ควรดูเหมือนเว็บดูดวง และไม่ควรดูเหมือนเว็บพนันหรือเว็บคริปโต

ความรู้สึกที่ต้องการคือ:

> "นี่คือระบบวิเคราะห์ธุรกิจและตัวตนที่จริงจัง แต่เข้าใจคนธรรมดาที่อยากมีเงินมากขึ้น"

ผู้ใช้งานจำนวนมากต้องการมีรายได้มากขึ้น แต่เผชิญกับข้อมูลและทางเลือกจำนวนมาก เช่น:

- ขายของออนไลน์
- Dropshipping
- Reselling
- ฟรีแลนซ์
- รับทำงานด้วยเครื่องมือดิจิทัล
- Digital Product
- UGC
- นายหน้า
- สร้างเพจ
- ทำ YouTube
- เปิดแบรนด์
- รับยิงโฆษณา
- เปิดเอเจนซี
- สร้าง Software
- ลงทุน

ปัญหาที่แท้จริงไม่ใช่ "ไม่มีวิธีหาเงิน"

แต่คือ:

> ผู้ใช้ไม่รู้ว่าเส้นทางใดเหมาะกับสถานการณ์ ความสามารถ และข้อจำกัดของตัวเอง

ผลที่เกิดขึ้นคือ:

1. ดูคอนเทนต์หาเงินจำนวนมาก แต่ไม่เคยเริ่มจริง
2. เปลี่ยนโมเดลรายได้ทุกสองสัปดาห์
3. ซื้อคอร์สหลายชุด แต่ไม่มีระบบเลือกว่าจะใช้ชุดใด
4. เลือกเส้นทางจากรายได้ของคนอื่น ไม่ได้เลือกจากความเหมาะสมของตัวเอง
5. ลงทุนผิดเกม เช่น คนไม่ชอบออกกล้องแต่ฝืนสร้าง Personal Brand
6. คนไม่มีเงินทุนไปเริ่มธุรกิจที่ต้องสต็อกสินค้า
7. คนต้องการเงินเร็ว แต่ไปสร้างธุรกิจที่ต้องรอผลหนึ่งถึงสองปี
8. คนไม่ชอบขาย แต่เลือกโมเดลที่ต้องโทรปิดการขายตลอดวัน
9. คนมีทักษะแต่ไม่รู้ว่าจะจัดทักษะเป็นสินค้าอย่างไร
10. คนไม่สามารถแยก "โอกาสจริง" ออกจาก "กระแสขายฝัน"

# 3. งานที่ผู้ใช้จ้างผลิตภัณฑ์ให้ทำ

## Functional Job
"ช่วยบอกว่าฉันควรเริ่มหาเงินจากอะไร และต้องทำอะไรเป็นลำดับแรก"

## Emotional Job
"ช่วยทำให้ฉันมั่นใจว่าฉันไม่ได้กำลังเสียเวลาไปผิดทางอีกครั้ง"

## Social Job
"ช่วยทำให้ฉันรู้สึกว่าฉันเป็นคนที่กำลังสร้างอนาคต ไม่ใช่คนที่ได้แต่บ่นว่าเงินไม่พอ"

## Hidden Job
ผู้ใช้ไม่ได้ต้องการเพียงแผนหาเงิน แต่ต้องการคนหรือระบบที่:

- ตัดตัวเลือกให้เหลือน้อยลง
- ให้ความหวังที่ดูมีเหตุผล
- ลดความกลัวว่าจะเลือกผิด
- ทำให้การเริ่มต้นดูง่าย
- สร้างความรู้สึกว่าผลลัพธ์นี้ออกแบบมาเพื่อตัวเอง
- บอกว่าควรหยุดทำอะไร
- ให้สิ่งที่สามารถเริ่มทำได้ภายในวันนี้

# 5. กลุ่มเป้าหมายหลัก

## กลุ่ม A: มนุษย์เงินเดือนที่อยากมีรายได้ทางที่สอง
ลักษณะ:

- อายุประมาณ 23–40 ปี
- มีงานประจำ
- มีเวลาหลังเลิกงานหรือวันหยุด
- ไม่มั่นใจว่าตัวเองเก่งพอจะขายอะไร
- กลัวลาออก
- ต้องการเงินเพิ่มเดือนละประมาณ 5,000–30,000 บาท
- สนใจ Affiliate, ฟรีแลนซ์ และขายของออนไลน์
- ซื้อความรู้มาแล้วแต่ไม่ลงมือ

สิ่งที่กลัว:

- เสียเงิน
- ถูกคนรู้จักหัวเราะ
- เริ่มแล้วไม่มีลูกค้า
- ลงแรงหลายเดือนแต่ไม่ได้เงิน

## กลุ่ม B: คนที่อยากออกจากงานประจำ
ลักษณะ:

- รู้สึกว่ารายได้ตัน
- ต้องการธุรกิจหรือรายได้ที่โตได้
- มักถามว่า "ควรเริ่มธุรกิจอะไรดี"
- มีเงินเก็บบางส่วนแต่ไม่อยากเสี่ยงทั้งหมด
- พร้อมจ่ายเพื่อให้ได้แผนที่ชัดเจน

## กลุ่ม C: ฟรีแลนซ์หรือเจ้าของกิจการรายย่อยที่รายได้ไม่แน่นอน
ลักษณะ:

- มีทักษะอยู่แล้ว
- ยังขายเป็นครั้ง ๆ
- ไม่มี Productized Service
- ไม่มีรายได้ประจำ
- อยากขยายจากแรงงานตัวเองไปเป็นระบบหรือสินค้า

## กลุ่ม D: คนที่ติดตามคอนเทนต์หาเงิน
ลักษณะ:

- ดูคลิป "รายได้เสริม" เป็นประจำ
- เซฟคลิปไว้จำนวนมาก
- สนใจทุกโมเดล แต่ไม่ลงมือจริง
- มีแรงปรารถนาสูง แต่ขาดระบบตัดสินใจ
- เป็นกลุ่มที่ตอบสนองต่อ FOMO และ Opportunity Content ได้ดี

# 8. โครงสร้างผลิตภัณฑ์ทั้งหมด

## ชั้นที่ 1: Free Money Scan
ผู้ใช้ตอบคำถามประมาณ 24–32 ข้อ

ได้รับฟรี:

- Money Type หลัก
- จุดแข็งในการสร้างรายได้
- จุดที่มักทำให้ผู้ใช้ไปไม่รอด
- เส้นทางที่เหมาะอันดับหนึ่งแบบย่อ
- คะแนนสำคัญ 3–5 ด้าน
- Preview ของรายงานฉบับเต็ม

เป้าหมายธุรกิจ:

- เก็บ Lead
- สร้างความรู้สึกว่า "ระบบเข้าใจฉัน"
- พาผู้ใช้ไปซื้อ Income Blueprint

รายงานฉบับเต็มแบบเสียเงิน

เนื้อหาประกอบด้วย:

1. Money Type หลักและรอง
2. สภาพการเงินและข้อจำกัดในการเริ่มต้น
3. สไตล์การสร้างรายได้
4. เส้นทางที่เหมาะ 3 อันดับ
5. เส้นทางที่ไม่เหมาะ 3 อันดับ
6. วิธีทำเงินของแต่ละเส้นทาง
7. ความเร็วในการเห็นรายได้
8. เงินทุนเริ่มต้นโดยประมาณ
9. ความเสี่ยง
10. ทักษะที่ต้องมี
11. วิธีเริ่มโดยใช้สิ่งที่มีอยู่
12. แผนทดลอง 7 วัน
13. แผน 30 วัน
14. เป้าหมายแรกที่ควรตั้ง
15. เกณฑ์ตัดสินว่าไปต่อหรือหยุด
16. สินค้าหรือบริการที่ผู้ใช้สามารถขายได้
17. ข้อความเสนอขายเบื้องต้น
18. คอนเทนต์เริ่มต้น
19. เครื่องมือที่ต้องใช้
20. ข้อควรระวังเฉพาะบุคคล

> **วิธีสร้างรายงาน (ไม่ใช้ AI):** เนื้อหาทั้งหมดประกอบขึ้นจาก **Content Blocks / Template** ที่ผูกกับ Money Type, Route และ Constraint Flags ที่ระบบคำนวณได้ (ดูหัวข้อ 17 และ 30) โดย Admin เป็นผู้เขียนและดูแลเทมเพลตเหล่านี้

# 9. โครงสร้างแบบประเมิน (Deterministic เท่านั้น)

ระบบใช้ **Deterministic Scoring** เพียงชั้นเดียว ไม่มีชั้น AI Personalization

## Deterministic Scoring
ใช้กฎและค่าน้ำหนักที่กำหนดไว้ชัดเจน

ระบบต้องคำนวณ Money Type และ Route ด้วยกฎที่ตรวจสอบได้ เพราะทำให้:

- ผลลัพธ์สม่ำเสมอ
- ทดสอบได้
- อธิบายสาเหตุได้
- ไม่มีต้นทุน API
- ผู้ใช้ตอบเหมือนเดิมได้ผลเหมือนเดิมเสมอ

## การประกอบเนื้อหารายงาน (แทนที่ AI Personalization)
หลังระบบคำนวณคะแนนแล้ว ระบบจะประกอบรายงานจาก **Template + Content Blocks** ที่ Admin จัดการ:

- เลือก Content Block ตาม Money Type / Route / Constraint Flags
- เติมค่าตัวแปร (เช่น ช่วงราคา ช่วงเวลา) จากผลคำนวณ
- แสดง Anti-route และ Bridge Route ตามกฎ
- ทุกอย่างเป็น deterministic และ reproducible

# 10. มิติที่ระบบต้องวัด

## Dimension A: Capital Readiness
วัด:

- เงินที่พร้อมเสียได้
- ความสามารถในการสต็อกสินค้า
- ความสามารถในการซื้อเครื่องมือ
- ความต้องการเริ่มแบบไม่ลงทุน

คะแนนต่ำเหมาะกับ: บริการ, Affiliate, นายหน้า, Content, Lead Generation
คะแนนสูงเหมาะกับ: แบรนด์สินค้า, Commerce, Software, ทีม, สินค้าคงคลัง

## Dimension B: Time Availability
วัด: เวลาต่อวัน, ความต่อเนื่องของเวลาว่าง, ทำได้เฉพาะวันหยุดหรือไม่, สามารถตอบลูกค้าระหว่างวันได้หรือไม่

## Dimension C: Income Urgency
วัด: ต้องการได้เงินเร็วแค่ไหน, สามารถรอสร้างสินทรัพย์ได้หรือไม่, มีรายได้หลักรองรับหรือไม่

- Urgency สูงเหมาะกับ: งานบริการ, Reselling, Commission, Freelance, Lead Generation
- Urgency ต่ำเหมาะกับ: Content Asset, Brand, Software, Digital Asset, Community

## Dimension D: Sales Tolerance
วัด: ความกล้าทักหาลูกค้า, ความสามารถในการเสนอราคา, ความรู้สึกต่อการถูกปฏิเสธ, ความชอบในการพูดคุย

## Dimension E: Visibility Tolerance
วัด: ยอมออกกล้องหรือไม่, ยอมใช้เสียงหรือไม่, ยอมใช้ชื่อจริงหรือไม่, กลัวคนรู้จักเห็นหรือไม่

## Dimension F: Skill Leverage
วัด: มีทักษะขายได้ทันทีหรือไม่, มีประสบการณ์เฉพาะทางหรือไม่, สร้างผลงานตัวอย่างได้หรือไม่, สามารถแก้ปัญหาให้ธุรกิจได้หรือไม่

## Dimension G: Operational Discipline
วัด: ความสม่ำเสมอ, ความสามารถทำงานซ้ำ, การติดตามตัวเลข, การจัดการรายละเอียด, ความอดทนต่อระบบหลังบ้าน

## Dimension H: Creative Production
วัด: การเขียน, การทำภาพ, การเล่าเรื่อง, การถ่ายวิดีโอ, การหา Angle, การทดลอง Hook

## Dimension I: Risk Tolerance
วัด: ความพร้อมเสียเงิน, ความพร้อมรับรายได้ไม่แน่นอน, ความพร้อมรับผลลัพธ์ช้า, ความมั่นคงของรายได้หลัก

## Dimension J: Scale Ambition
วัด: ต้องการเพียงรายได้เสริม, ต้องการออกจากงาน, ต้องการธุรกิจที่มีทีม, ต้องการทรัพย์สินที่ขายต่อได้, ต้องการรายได้สูงแม้ต้องใช้เวลานาน

# 12. Money Types

Money Type คือ "รูปแบบการทำเงินตามธรรมชาติ" ไม่ใช่อาชีพ

ผู้ใช้หนึ่งคนมี: Primary Money Type, Secondary Money Type, Anti-Type (ประเภทที่ฝืนธรรมชาติ)

## Type 1: THE HUNTER
**ลักษณะ:** ต้องการเห็นเงินเร็ว, กล้าทัก กล้าขาย กล้าปิด, รับการปฏิเสธได้, ชอบดีลมากกว่าการสร้างระบบยาว, มีพลังเมื่อเห็นผลลัพธ์
**เส้นทางที่เหมาะ:** Affiliate, นายหน้า, Commission Sales, Reselling, Deal Sourcing, Lead Generation, Appointment Setting
**จุดอ่อน:** เบื่องานสร้างสินทรัพย์ระยะยาว, เปลี่ยนสินค้าบ่อย, ขึ้นกับแรงและโอกาสใหม่, อาจให้ความสำคัญกับยอดขายมากกว่าคุณภาพ

## Type 2: THE CREATOR
**ลักษณะ:** เก่งคิด Hook, สร้างคอนเทนต์ได้, เข้าใจ Attention, เปลี่ยนไอเดียเป็นภาพหรือเรื่องเล่าได้, มีโอกาสสร้าง Audience
**เส้นทางที่เหมาะ:** Faceless Media, UGC, Affiliate Content, Content Product, Digital Media Asset, Channel Building, Creative Service
**จุดอ่อน:** สนุกกับการเริ่ม แต่เบื่อระบบ, ผลิตมากแต่ไม่วัด Conversion, สร้างผู้ติดตามแต่ไม่มีข้อเสนอขาย

## Type 3: THE EXPERT
**ลักษณะ:** มีทักษะหรือความรู้เฉพาะ, แก้ปัญหาให้ผู้อื่นได้, เหมาะกับข้อเสนอราคาสูง, สร้างความน่าเชื่อถือผ่านผลงาน
**เส้นทางที่เหมาะ:** Freelance, Consulting, Productized Service, Digital Product, Workshop, Training, B2B Service
**จุดอ่อน:** เรียนเพิ่มตลอดแต่ไม่ขาย, ทำงานเฉพาะตัวมากเกินไป, รายได้ติดอยู่กับเวลา

## Type 4: THE OPERATOR
**ลักษณะ:** ทำตามระบบได้ดี, ชอบจัดการรายละเอียด, ทำให้งานหลังบ้านไหลลื่น, เหมาะกับบริการที่ลูกค้าไม่อยากทำเอง
**เส้นทางที่เหมาะ:** Virtual Assistant, Marketplace Operations, Content Operations, CRM Management, Automation Service, Ads Operations, E-commerce Operations
**จุดอ่อน:** มักตั้งราคาต่ำ, ซ่อนตัวอยู่หลังงาน, ไม่สร้างข้อเสนอที่มีมูลค่าสูง

## Type 5: THE MERCHANT
**ลักษณะ:** เข้าใจสินค้า ราคา และความต้องการ, ชอบหา Margin, สนใจสิ่งที่ขายได้จริงมากกว่าทฤษฎี, สามารถจัดการ Supply และ Demand
**เส้นทางที่เหมาะ:** Reselling, Marketplace, TikTok Shop, Private Label, Niche Commerce, Pre-order, Product Curation
**จุดอ่อน:** เงินจมในสต็อก, แข่งขันราคา, พึ่งแพลตฟอร์มมากเกินไป, เลือกสินค้าจากความชอบแทน Demand

## Type 6: THE BUILDER
**ลักษณะ:** ยอมรอเพื่อสร้างทรัพย์สิน, สนใจระบบและการขยาย, ต้องการสร้างสิ่งที่ทำงานได้โดยไม่ขึ้นกับแรงตลอดเวลา, มีความทะเยอทะยานระยะยาว
**เส้นทางที่เหมาะ:** Brand, SaaS, Application, Membership, Marketplace, Content Library, Intellectual Property, Subscription Business
**จุดอ่อน:** สร้างนานก่อนคุยกับตลาด, Overbuild, ใช้เงินกับระบบก่อนมีลูกค้า, มองข้าม Cashflow ระยะสั้น

# 13. Actual Income Routes

Money Type กับ Income Route ต้องแยกออกจากกัน

ตัวอย่าง: ผู้ใช้เป็น Creator แต่เส้นทางที่เหมาะอาจเป็น Faceless Affiliate หรือ UGC Service; ผู้ใช้เป็น Builder แต่เนื่องจากมีเงินทุนน้อย ระบบอาจแนะนำให้เริ่ม Productized Service ก่อน แล้วค่อยขยับไป Software

## Route 1: Commission Route
**รูปแบบ:** Affiliate, นายหน้า, Referral, Appointment Setting, Commission Sales
**เหมาะกับ:** เงินทุนน้อย, ต้องการรายได้เร็ว, ยอมขาย, ไม่จำเป็นต้องสร้างสินค้าของตัวเอง

## Route 2: Faceless Content Route
**รูปแบบ:** TikTok Faceless, YouTube Faceless, Affiliate Content, Theme Page, Curated Media
**เหมาะกับ:** ไม่อยากออกหน้า, ทำคอนเทนต์ได้, มีความสม่ำเสมอ, ยอมรอสะสม Audience

## Route 3: Productized Service Route
**รูปแบบ:** บริการที่กำหนด Scope ราคา และผลลัพธ์ชัดเจน เช่น แพ็กเกจตัดคลิป แพ็กเกจทำภาพ
**เหมาะกับ:** มีทักษะ, ต้องการเงินเร็ว, ไม่อยากรับงานแบบตามใจลูกค้าทุกอย่าง

## Route 4: Digital Service Route
**รูปแบบ:** Content Production, Catalog, Presentation, Automation Setup, Customer Support Setup (ใช้เครื่องมือดิจิทัลทั่วไป)
**เหมาะกับ:** ใช้เครื่องมือดิจิทัลได้, เรียนรู้เร็ว, เข้าใจปัญหาธุรกิจ, สร้างตัวอย่างงานได้

## Route 5: Reselling Route
**รูปแบบ:** ของมือสอง, Marketplace Arbitrage, สินค้าพรีออเดอร์, สินค้าหายาก, ซื้อมาแบ่งขาย
**เหมาะกับ:** ชอบสินค้า, มองราคาออก, มีพื้นที่หรือระบบส่งของ, ต้องการเงินหมุนเร็ว

## Route 6: Local Business Route
**รูปแบบ:** รับดูแล Google Business Profile, รับทำเมนู, รับทำคอนเทนต์, รับสร้าง Landing Page, รับจัดระบบ LINE OA, Lead Generation
**เหมาะกับ:** เข้าถึงร้านค้าได้, กล้าคุยกับเจ้าของกิจการ, ไม่จำเป็นต้องมีผู้ติดตาม

## Route 7: Digital Asset Route
**รูปแบบ:** Template, Toolkit, Database, Membership, Calculator, Mini Application, Digital Product
**เหมาะกับ:** สร้างระบบได้, มีความรู้เฉพาะ, ยอมรอเพื่อสร้างสินทรัพย์, ทำการตลาดออนไลน์ได้

## Route 8: Commerce Brand Route
**รูปแบบ:** Private Label, Niche Brand, Meaningful Product, Subscription Product, Creator Commerce
**เหมาะกับ:** มีทุน, ยอมจัดการสินค้า, ต้องการสร้างแบรนด์, มีระยะเวลารอ

## Route 9: Agency Route
**รูปแบบ:** รวมบริการหลายอย่าง, ใช้ทีมส่งมอบ, รายได้ประจำจากลูกค้าองค์กร
**เหมาะกับ:** ขายเก่ง, บริหารคนได้, มีระบบ, รับแรงกดดันจากลูกค้าได้

## Route 10: Software Route
**รูปแบบ:** Micro SaaS, Vertical Software, Calculator, Workflow Tool
**เหมาะกับ:** คิดเชิงระบบ, มีทักษะหรือ Partner ด้านเทคนิค, ยอมทำ Product Discovery, ยอมรอรายได้มากกว่าเส้นทางบริการ

# 14. Logic การให้คะแนน

## หลักการ
แต่ละคำตอบเพิ่มหรือลดคะแนนใน: Dimensions, Money Types, Income Routes, Risk Flags

ตัวอย่าง ผู้ใช้ตอบว่า: ทุนน้อย, ต้องการเงินภายใน 30 วัน, ไม่ออกหน้า, กล้าทักลูกค้า, มีทักษะทำคอนเทนต์

ระบบอาจคำนวณว่า:
- Hunter: 78 / Creator: 72 / Expert: 55 / Operator: 48 / Merchant: 42 / Builder: 37

Route Match:
- Productized Creative Service: 88 / Faceless Affiliate: 83 / Lead Generation: 74 / Digital Product: 61 / Commerce Brand: 29

## สูตรพื้นฐาน
สำหรับแต่ละ Route:

`Route Score = Dimension Match + Type Match + Resource Match - Constraint Penalty + Strategic Bonus`

ตัวอย่างน้ำหนัก:
- Dimension Match 50%
- Money Type Match 20%
- Resource Match 20%
- Constraint Penalty สูงสุด -30%
- Strategic Bonus สูงสุด +10%

## ตัวอย่าง Constraint Penalty
หากผู้ใช้ต้องการได้เงินภายใน 30 วัน: Software Route -20, Content Asset Route -15, Brand Route -15
หากผู้ใช้ไม่ยอมขายและไม่มีผู้ติดตาม: Affiliate Route -10, Consulting Route -20, Agency Route -20
หากผู้ใช้มีทุนต่ำกว่า 5,000 บาท: Inventory Commerce -25, Physical Brand -20, SaaS ที่ต้องจ้างพัฒนา -20

## กฎสำคัญ
1. อย่าแนะนำเส้นทางจากคะแนนสูงเพียงอย่างเดียว
2. ต้องตรวจ Constraint ก่อน
3. ผู้ใช้ต้องได้รับคำแนะนำที่เริ่มได้จากสถานการณ์จริง
4. หาก Route ที่เหมาะระยะยาวไม่ตอบ Income Urgency ต้องสร้าง Bridge Route

ตัวอย่าง:
> เส้นทางระยะยาวของคุณคือ Software Builder แต่ในช่วงแรกควรใช้ Productized Service สร้าง Cashflow และเรียนรู้ปัญหาลูกค้าก่อน

นี่คือหนึ่งในจุดที่ทำให้ระบบแตกต่างจากแบบทดสอบทั่วไป

# 15. Bridge Route

Bridge Route คือเส้นทางชั่วคราวที่ช่วยให้ผู้ใช้: มีเงินสด, เรียนรู้ตลาด, สร้างลูกค้า, สร้าง Portfolio, เก็บข้อมูล, ลดความเสี่ยงก่อนสร้างธุรกิจระยะยาว

| Destination Route | Bridge Route |
|---|---|
| Software | รับทำระบบหรือ Automation ให้ลูกค้าก่อน |
| Digital Product | รับ Consulting หรือ Freelance ก่อน |
| Physical Brand | Affiliate หรือ Reselling ในหมวดเดียวกันก่อน |
| Agency | เริ่ม Productized Service หนึ่งรายการ |
| Content Business | เริ่ม UGC หรือ Affiliate Content |
| Membership | เริ่ม Newsletter หรือ Community ฟรี |

ผลลัพธ์ของผู้ใช้ต้องแสดงทั้ง: Cashflow Route, Asset Route, Bridge Strategy

# 26. Ethical Persuasion ที่ใช้ได้

- **Personalization:** แสดงผลตามข้อมูลจริงของผู้ใช้
- **Partial Reveal:** ให้ผลฟรีที่มีคุณค่า แต่เก็บรายละเอียดเชิงปฏิบัติไว้ในฉบับเต็ม
- **Loss Framing:** "ทุกเดือนที่คุณเปลี่ยนเส้นทางโดยไม่ทดลองให้ครบ คุณไม่ได้เสียแค่เวลา แต่เสียโอกาสในการสะสมทักษะและข้อมูล" (ไม่สร้างตัวเลขรายได้ที่เสียไปโดยไม่มีสมมติฐาน)
- **Identity:** "จากคนที่กำลังหาโอกาส สู่คนที่กำลังทดลองโอกาสอย่างเป็นระบบ"
- **Real Scarcity:** ใช้เฉพาะกรณีจริง (Challenge จำกัดที่นั่ง, Promotion มีวันจบจริง, Bonus มี Capacity จริง) ห้าม Countdown ปลอมหรือที่นั่งปลอม

- Tailwind CSS

App Router รองรับ Routing ตามโครงสร้างไฟล์ รวมถึง Server Components, Suspense และ Server Functions จึงเหมาะกับแอปที่มีทั้ง Landing Page, Assessment, Dashboard และ Server-side Logic ในโปรเจกต์เดียว

- Supabase Auth
- Row Level Security
- Supabase Storage
- Edge Functions หรือ Server Routes
- Scheduled Jobs สำหรับสมาชิกและ Opportunity Update

Supabase ให้ Postgres เป็นฐานข้อมูลหลัก และ Auth เชื่อมกับ RLS เพื่อจำกัดข้อมูลระดับผู้ใช้ ส่วน Edge Functions เหมาะกับ Webhook และงาน Server-side ที่ต้องเก็บ Secret ไว้นอก Client

- Email + Password
- Magic Link
- Google Login เป็นตัวเลือก
- Guest Assessment ก่อน Register
- เมื่อจบ Assessment ค่อยสร้างบัญชีหรือ Claim Session

- บัตร, PromptPay, Coupon
- One-time Purchase, Subscription

## Report Engine (แทน AI Layer — ไม่ใช้ AI)
- สร้างรายงานด้วย **Deterministic Template Engine** ฝั่ง Server
- ประกอบเนื้อหาจาก **Content Blocks** ที่ผูกกับ Money Type / Route / Constraint Flags
- เติมค่าตัวแปร (ช่วงราคา, ช่วงเวลา, ชื่อ Route) จาก score snapshot
- ผลลัพธ์เป็น reproducible: input เดียวกันได้รายงานเดียวกันเสมอ
- บันทึก `report_version` และ `template_version` ทุกครั้ง
- มี Fallback Template หากไม่มี Content Block ตรงกับเงื่อนไข
- Validate โครงสร้าง output ด้วย Zod ก่อนบันทึก
- ห้าม Render HTML ที่ไม่ผ่านการ Sanitize

## Privacy
ข้อมูลแบบประเมินอาจสะท้อน: รายได้, เงินทุน, ภาระทางการเงิน, พฤติกรรม, เป้าหมายส่วนบุคคล

> เนื่องจากไม่มีการส่งข้อมูลออกไปยังบริการ AI ภายนอก ความเสี่ยงด้าน Data Sharing จึงลดลงมาก ข้อมูลผู้ใช้ถูกประมวลผลภายในระบบของเราเท่านั้น

# 30. Structured Output ของรายงาน (Deterministic Template)

> โครงสร้าง JSON นี้ยังใช้เป็น **สัญญาข้อมูลของรายงาน (contract)** แต่เนื้อหาถูกเติมโดย **Template Engine** ไม่ใช่ AI
> ทุกฟิลด์ต้องมาจาก Content Block ที่ Admin เขียนไว้ + ค่าจาก score snapshot และต้องผ่าน Zod validation

```json
{
  "executive_summary": {
    "headline": "string",
    "diagnosis": "string",
    "primary_advantage": "string",
    "primary_constraint": "string",
    "recommended_next_move": "string"
  },
  "money_pattern": {
    "primary_type": "string",
    "secondary_type": "string",
    "anti_type": "string",
    "behavioral_insights": ["string"],
    "common_failure_modes": ["string"]
  },
  "primary_route": {
    "route_name": "string",
    "fit_score": 0,
    "why_it_fits": ["string"],
    "how_money_is_made": "string",
    "ideal_customer": "string",
    "starter_offer": {
      "name": "string",
      "deliverables": ["string"],
      "starter_price_range": "string",
      "proof_required": ["string"]
    },
    "risks": ["string"]
  },
  "secondary_routes": [
    { "route_name": "string", "fit_score": 0, "role_in_strategy": "string" }
  ],
  "anti_routes": [
    { "route_name": "string", "reason": "string", "reconsider_when": "string" }
  ],
  "bridge_strategy": {
    "cashflow_route": "string",
    "asset_route": "string",
    "transition_condition": "string"
  },

    { "day": 1, "objective": "string", "actions": ["string"], "completion_criteria": "string" }
  ],

    "week_1": ["string"],
    "week_2": ["string"],
    "week_3": ["string"],
    "week_4": ["string"]

  "disclaimer": "string"

```

## หลักการ Template Engine
1. เลือก Content Block ต่อ section ตาม `condition` ที่ match กับ primary_type / secondary_type / primary_route / constraint_flags
2. เรียงลำดับความเฉพาะเจาะจง (specific > general) และเลือกอันที่ match ดีที่สุด
3. เติมตัวแปร (`{{route_name}}`, `{{starter_price_range}}` ฯลฯ) จาก score snapshot
4. หากไม่มี Block ตรงเงื่อนไข ใช้ Fallback Block ระดับ Route/Type ทั่วไป
5. Validate ด้วย Zod → หากไม่ผ่าน บันทึกสถานะ error และไม่แสดงรายงานที่ไม่สมบูรณ์

> เดิมเป็น "AI System Prompt" — ตัดออกและแปลงเป็น **แนวทางเขียน Content Block สำหรับ Admin**

แนวทางเขียนเนื้อหาในทุก Content Block:

**โทนแบรนด์:** ตรง, ฉลาด, สุขุม, ปฏิบัติได้จริง, ซื่อสัตย์เรื่อง trade-off, ไม่ร่าเริงเกินจริง, ไม่ตำหนิผู้ใช้, ไม่ใส่คำ motivational กลวง ๆ, ไม่สัญญาความรวยหรือรายได้ที่การันตี, เขียนภาษาไทยที่คนทั่วไปเข้าใจ

**ข้อกำหนดของเนื้อหา:**
1. เคารพเงื่อนไขจริงของผู้ใช้ (ทุน เวลา visibility sales tolerance urgency) ผ่านการ match condition
2. อธิบายว่าทำไมคำแนะนำจึงเข้ากับคะแนนที่คำนวณได้
3. แนะนำการทดลองที่เป็นรูปธรรมก่อนการลงทุนใหญ่
4. แยก Fast Cashflow Route ออกจาก Long-term Asset Route
5. ระบุ Route ที่ควรหลีกเลี่ยงในตอนนี้
6. ใช้ภาษาที่สมจริง
7. หลีกเลี่ยงคำแนะนำการลงทุนเฉพาะบุคคล
8. หลีกเลี่ยงวิธีผิดกฎหมาย หลอกลวง ละเมิดลิขสิทธิ์ หลบเลี่ยงแพลตฟอร์ม
9. ไม่ใส่รีวิวปลอม หลักฐานรายได้ปลอม ความเร่งด่วนปลอม
10. ราคาให้ระบุเป็น "ช่วงราคาทดสอบเริ่มต้น" ไม่ใช่ราคาตลาดที่การันตี
11. Timeline ให้กรอบเป็น "ช่วงทดสอบ / milestone" ไม่ใช่กำหนดรายได้ที่การันตี
